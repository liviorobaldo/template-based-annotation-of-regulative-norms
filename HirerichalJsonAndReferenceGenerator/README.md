## Deontic Context Annotator — Hierarchical JSON and Reference Generator

### Overview
This folder contains two standalone Python scripts:
- `json_legislion_builder.py`: Parses a LegalDocML XHTML file and generates a hierarchical JSON of Parts, Chapters, Sections, and nested items.
- `reference_assigner.py`: Reads the JSON produced by the builder and adds regex-detected cross-references to sections.

### Input/Output Relationship
- **Builder input (`json_legislion_builder.py`)**: An XHTML file generated by the annotation script in the parent directory. Place XHTML files under `processed_acts/` or pass an absolute path.
- **Builder output**: A JSON file capturing the act hierarchy. Move/copy this JSON into `input_for_reference_assigner/` and ensure the filename ends with `_sections_hierarchy.json`.
- **Assigner input (`reference_assigner.py`)**: The builder’s JSON from `input_for_reference_assigner/` (expects suffix `_sections_hierarchy.json`).
- **Assigner output**: Augmented JSON with references written to `outputs_for_reference_assigner/regex/`.

### Prerequisites
- Python 3.x
- Required Python packages:
  - `requests`
  - `beautifulsoup4`
  - `lxml`

### Installation
1. Clone the repository
2. Install required packages (no virtual environment is required for these steps):
```bash
pip install -r requirements.txt
# or
python3 -m pip install -r requirements.txt
```

### Running the Scripts
- Run the Builder (generate hierarchical JSON from XHTML):
```bash
python3 json_legislion_builder.py
```

- Then run the Reference Assigner (augment JSON with regex references):
```bash
python3 reference_assigner.py
```

### Notes
- Both scripts are designed to run standalone.
- The builder’s input XHTML comes from the annotation script in the parent directory; please refer there for how those XHTML files are generated.
- Ensure builder output JSON filenames end with `_sections_hierarchy.json` so the assigner can pick them up automatically.
