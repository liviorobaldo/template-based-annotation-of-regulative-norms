## Obligations, Permissions & Prohibitions Annotation Matcher

This pipeline assigns each annotator-written obligation/permission/prohibition card to its source section so ground-truth datasets stay aligned. It:
- Consumes per-part annotation text exported by annotators
- Loads the matching hierarchical JSON generated by `HirerichalJsonAndReferenceGenerator`
- Maps every annotation block to the best section candidate
- Emits aligned annotations for analysis or downstream systems

Inputs are configured near the bottom of `annotation_matcher.py`:
- `annotation_file`: Place the path for annotation file
- `section_json_file`: place the hierarchical act JSON under the same directory 
- `url`: base legislation.gov.uk URL corresponding to that act/part

Outputs are written next to the inputs:
- JSON/CSV/XML files under `<Act>/outputs/` containing matched section URLs and condition details

## Getting Started

### Prerequisites

- Python 3.9+
- Required packages (install via `pip install -r requirements.txt`)

### Prepare Inputs

1. Run `HirerichalJsonAndReferenceGenerator` to create the per-part hierarchical JSON and store it under `<Act>/`.
2. Place the matching annotator file (same part) next to it using the `ChunkN.txt` naming pattern.

### Running the Pipeline

```bash
python annotation_matcher.py
# or
python3 annotation_matcher.py
```

Update the file paths in the `__main__` block before running a different part.

### Aggregating Annotations Per Section

After generating the chunked CSV files in `Children Act 1989/outputs/`, you can collapse every annotation down to the section(s) it references by running:

```bash
python3 aggregate_section_annotations.py \
  "Children Act 1989/outputs" \
  "Children Act 1989/1989_41_sections_hierarchy.json" \
  "https://www.legislation.gov.uk/ukpga/1989/41" \
  "Children Act 1989/outputs/sections_combined.csv"
```

You can also run `python3 aggregate_section_annotations.py` with no arguments to use those default paths.

The aggregated CSV uses the following columns:

- `input`: a single `section-*` identifier per row.
- `section_text`: JSON containing the section identifier and text extracted from the hierarchy JSON.
- `output_json`: JSON list of every annotation record (from the original matcher output) that references the section.
- `output_text`: JSON list of raw annotation card text for that section.
